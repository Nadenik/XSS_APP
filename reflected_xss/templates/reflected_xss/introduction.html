Introduction

This type of Cross Site Scripting attack(XSS) is one in which HTML/JS code contained in any query parameter (GET, POST or even cookies) is then displayed in the response. It is said that the value of the parameter has been reflected in the response, hence the name „Reflected XSS".
For instance in following popular search request:
http://www.randomsite.com/search=our_arbitrary_query
Search parameter is being set by an user and it could be anything. If we don't sanitize the input then the following request:
http://www.randomsite.com/search=<script>alert(1)</script>
Would execute alert(1) function.

The practical use of Reflected XSS usually involves sending a the victim a suitably crafted link (e.g. by e-mail or instant messaging) containing malicious code.
